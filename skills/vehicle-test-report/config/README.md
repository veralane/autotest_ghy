# 测试要求配置说明

## 概述

本配置文件用于定义各类测试项目的测试要求值，用户可以根据实际需求修改这些要求。

## 配置文件位置

```
config/test_requirements.json
```

## 配置结构

每个测试项目包含以下字段：

### 1. test_name
测试项目名称，用于报告显示。

### 2. requirements
测试要求值，定义每个测试指标的通过标准。

**格式：**
```json
{
  "requirements": {
    "指标名称": "要求值",
    "平均减速度(m/s²)": "≥8.0",
    "制动距离(m)": "≤40"
  }
}
```

**要求值格式：**
- `≥X` - 大于等于X
- `≤X` - 小于等于X
- `X-Y` - 在X到Y之间
- `通过/失败` - 定性判断

### 3. default_values
默认测试条件值，用于生成报告模板时填充。

**格式：**
```json
{
  "default_values": {
    "附着系数": 0.85,
    "测试路面": "干沥青路面",
    "测试温度": 22,
    "测试湿度": 55
  }
}
```

## 如何修改配置

### 方法1：直接编辑JSON文件

1. 打开 `config/test_requirements.json`
2. 找到对应的测试项目
3. 修改 `requirements` 中的值
4. 保存文件

**示例：修改干沥青直线制动的减速度要求**

```json
{
  "abs_straight_braking": {
    "requirements": {
      "平均减速度(m/s²)": "≥8.5",  // 从8.0改为8.5
      ...
    }
  }
}
```

### 方法2：通过技能命令修改

```
修改干沥青直线制动的减速度要求为≥8.5 m/s²
```

## 注意事项

1. **数值格式**
   - 使用标准数值格式（如：8.0、0.85）
   - 保持单位一致

2. **比较符号**
   - `≥` - 大于等于
   - `≤` - 小于等于
   - `>` - 大于
   - `<` - 小于

3. **单位说明**
   - m/s² - 米每平方秒
   - m - 米
   - deg - 度
   - s - 秒
   - % - 百分比

4. **配置生效**
   - 修改配置后，下次生成报告时自动生效
   - 已生成的报告不受影响

## 添加新测试项目

如需添加新的测试项目，按以下格式添加：

```json
{
  "new_test_item": {
    "test_name": "新测试项目名称",
    "requirements": {
      "指标1": "要求值1",
      "指标2": "要求值2"
    },
    "default_values": {
      "默认条件1": "值1",
      "默认条件2": "值2"
    }
  }
}
```
