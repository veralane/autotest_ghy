# 示例文件说明

本目录包含整车功能性能测试报告的示例文件，供用户参考。

## 文件列表

### 1. test_data.json

**测试数据示例（JSON格式）**

包含完整的测试数据结构，包括：
- 项目基本信息
- 测试版本信息
- 测试环境描述
- ABS测试结果（新格式）
  - 干沥青直线制动（50/80/100 km/h）
  - 干沥青弯道制动（50/80 km/h）
- TCS测试结果

**用途：**
- 作为报告生成的数据输入
- 了解测试数据结构定义
- 作为自定义测试数据的模板

### 2. complete_report_sample.docx

**完整测试报告示例（Word格式）**

包含完整5章内容：
- 第一章：前言
- 第二章：概述
- 第三章：测试版本说明
- 第四章：概要测试结论
- 第五章：测试项目及结果

**最新改进：**
- ✅ 路面附着系数列（每次测试填写）
- ✅ 测试要求行（表头下方，黄色高亮）
- ✅ 完整5章内容
- ✅ 所有表格格式规范

**用途：**
- 查看完整报告结构
- 了解各章节内容
- 作为正式报告模板

### 3. vehicle_test_report_sample.md

**完整测试报告示例（Markdown格式）**

与Word版本内容相同，但以Markdown格式呈现。

**用途：**
- 快速预览报告内容
- 在线查看和编辑
- 转换为其他格式

## 使用方法

### 查看示例报告

1. **Word格式**：直接打开 `vehicle_test_report_sample.docx`
2. **Markdown格式**：使用文本编辑器或Markdown查看器打开 `vehicle_test_report_sample.md`

### 使用示例数据生成报告

```bash
python scripts/generate_report.py --output my_report.docx --data examples/test_data.json
```

### 自定义测试数据

1. 复制 `test_data.json` 为 `my_test_data.json`
2. 修改测试数据内容
3. 使用修改后的数据生成报告

## 数据结构说明

### ABS测试结果（新格式）

```json
{
  "abs_straight_braking": {
    "test_conditions": {
      "测试路面": "干沥青路面",
      "路面附着系数": 0.85,
      "测试温度": 22,
      "测试湿度": 55
    },
    "test_items": [
      {
        "车速": 50,
        "test_runs": [
          { "序号": 1, "平均减速度": 8.6, ... },
          { "序号": 2, "平均减速度": 8.4, ... },
          { "序号": 3, "平均减速度": 8.5, ... }
        ],
        "average": { "平均减速度": 8.5, ... },
        "requirements": { "平均减速度": "≥8.0", ... }
      }
    ],
    "subjective_evaluation": "制动过程平稳..."
  }
}
```

### 表格字段说明

**新格式表格字段（v2.0）：**

| 字段 | 单位 | 说明 |
|------|------|------|
| 测试项目(车速) | km/h | 测试车速 |
| 测试序号 | - | 1、2、3或平均 |
| **路面附着系数** | - | **测试路面附着系数（新增）** |
| 平均减速度 | m/s² | 制动过程平均减速度 |
| 制动距离 | m | 制动距离 |
| 减速度峰谷差值 | m/s² | 减速度波动指标 |
| 转向修正角 | deg | 转向修正角度 |
| 车轮抱死时间 | s | 车轮抱死时间 |
| 附着系数利用率 | % | 附着系数利用率 |
| 主观评分 | 1-10分 | 主观评价 |
| 结论 | - | 通过/失败 |

**表格结构说明：**
- **第1行**：表头（蓝色背景）
- **第2行**：测试要求（黄色背景，斜体）
- **第3-N行**：测试数据
- **平均值行**：加粗显示

## 注意事项

1. **编码格式**：JSON文件使用UTF-8编码
2. **数值精度**：建议保留1-2位小数
3. **单位统一**：确保所有数据使用统一单位
4. **必填字段**：参考 `references/data_schema.md` 了解必填字段
