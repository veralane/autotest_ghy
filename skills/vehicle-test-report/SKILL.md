# 整车功能性能测试报告生成

## 概述

本技能用于生成整车功能性能测试报告，输出格式为Word文档(.docx)。报告遵循标准章节结构，适用于ABS、TCS等车辆功能测试的结果记录。

**核心特性：**
- ✅ 自动生成完整5章内容
- ✅ 表格包含路面附着系数列
- ✅ 表格包含测试要求行
- ✅ 支持用户自定义测试要求配置
- ✅ 中文格式输出

## 快速开始

**生成完整报告：**
```
生成一份整车功能性能测试报告
```

技能将自动输出包含5章内容的Word文档，无需额外配置。

## 报告章节结构

### 第一章 前言

简要说明报告目的、编写依据和适用范围。

**内容要点：**
- 报告编写目的
- 参考标准/规范
- 报告适用范围

### 第二章 概述

对测试对象和测试活动的总体描述。

**内容要点：**
- 测试对象描述（车型、配置等）
- 测试目标
- 测试周期
- 测试团队

### 第三章 测试版本说明

详细记录测试相关的版本和环境信息。

**内容要点：**

#### 3.1 测试版本信息
- 软件版本号
- 硬件版本
- 固件版本
- 版本变更记录

#### 3.2 测试环境描述
- 测试场地
- 测试设备
- 测试工具
- 环境条件（温度、湿度等）

#### 3.3 引用的测试设计
- 测试用例文档编号
- 测试规范文档
- 需求规格说明

#### 3.4 测试通过标准
- 功能通过准则
- 性能指标要求
- 安全性要求

### 第四章 概要测试结论

对测试结果的总体评价。

**内容要点：**

#### 4.1 测试结论总结
- 测试执行情况统计
- 通过/失败/阻塞用例数量
- 测试覆盖率
- 总体结论（通过/不通过/有条件通过）

#### 4.2 关键风险和规避措施
- 已识别风险列表
- 风险等级评估
- 规避措施说明
- 遗留问题跟踪

### 第五章 测试项目及结果

详细记录各测试项的执行结果。

## 使用方法

### 方式1：自动生成完整报告（推荐）

直接请求生成完整报告，技能将自动输出包含5章内容的Word文档：

```
生成一份整车功能性能测试报告
```

或

```
帮我生成XX车型的ABS/TCS测试报告
```

**输出内容：**
- ✅ 完整5章内容
- ✅ 所有表格（含路面附着系数列和测试要求行）
- ✅ 中文格式
- ✅ Word文档(.docx)

### 方式2：使用Python脚本生成

使用Python脚本 `scripts/generate_report.py` 生成报告：

```bash
python scripts/generate_report.py --output report.docx --data test_data.json
```

### 参数说明

- `--output`: 输出文件路径（默认：vehicle_test_report.docx）
- `--data`: 测试数据JSON文件路径（可选）
- `--template`: 自定义模板路径（可选）

### 测试数据格式

参考 `references/data_schema.md` 了解测试数据的JSON格式定义。

## 配置说明

### 测试要求配置

用户可通过修改 `config/test_requirements.json` 自定义测试要求：

**配置文件结构：**
```json
{
  "abs_straight_braking": {
    "test_name": "干沥青直线制动",
    "requirements": {
      "附着系数": "≥0.80",
      "平均减速度(m/s²)": "≥8.0",
      "制动距离(m)": "≤40",
      ...
    }
  }
}
```

**支持的测试项目：**
- `abs_straight_braking` - 干沥青直线制动
- `abs_curve_braking` - 干沥青弯道制动
