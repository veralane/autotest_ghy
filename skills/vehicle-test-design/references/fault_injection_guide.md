# 故障注入测试指南

## 一、故障注入测试概述

### 1.1 定义

故障注入测试（Fault Injection Testing）是一种通过人为引入故障来验证系统故障检测和处理能力的测试方法。

### 1.2 目的

1. 验证故障检测机制的有效性
2. 验证故障响应时间的符合性
3. 验证安全状态进入的正确性
4. 验证系统容错能力
5. 满足ISO 26262功能安全要求

### 1.3 适用场景

- ASIL B及以上等级的安全相关功能
- 具有故障检测机制的系统
- 具有冗余设计的系统
- 需要验证安全状态的系统

---

## 二、故障类型分类

### 2.1 按故障来源分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 硬件故障 | 硬件组件的物理故障 | 传感器失效、执行器卡滞、电源故障 |
| 软件故障 | 软件运行时的错误 | 数据越界、计算溢出、内存泄漏 |
| 通信故障 | 通信过程中的故障 | 信号丢失、信号延迟、信号错误 |
| 环境故障 | 外部环境引起的故障 | 温度超限、EMI干扰、湿度超限 |

### 2.2 按故障持续时间分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 永久故障 | 持续存在的故障 | 传感器完全损坏、线路断开 |
| 间歇故障 | 时有时无的故障 | 接触不良、温度敏感故障 |
| 瞬态故障 | 短暂出现的故障 | EMI干扰、电源波动 |

### 2.3 按故障影响分类

| 类型 | 说明 | 测试重点 |
|------|------|----------|
| 单点故障 | 单个组件故障导致的系统故障 | 故障检测能力、安全状态进入 |
| 潜伏故障 | 未被检测到的故障 | 故障检测覆盖率 |
| 多点故障 | 多个组件同时故障 | 冗余机制、共因分析 |
| 共因故障 | 同一原因导致的多个故障 | 冗余独立性验证 |

## 三、故障注入方法

### 3.1 硬件故障注入

#### 3.1.1 信号级注入

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 信号断开 | 断开传感器/执行器信号线 | 信号丢失测试 |
| 信号短路 | 将信号线短路到电源/地 | 短路故障测试 |
| 信号接电阻 | 在信号线上串联/并联电阻 | 信号衰减测试 |
| 信号叠加噪声 | 在信号上叠加噪声信号 | 抗干扰测试 |

#### 3.1.2 电源级注入

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 电源断开 | 切断供电 | 电源丢失测试 |
| 欠压注入 | 降低供电电压 | 欠压保护测试 |
| 过压注入 | 升高供电电压 | 过压保护测试 |
| 电源波动 | 注入电压波动 | 电源稳定性测试 |

#### 3.1.3 通信级注入

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 总线断开 | 断开通信总线 | 通信中断测试 |
| 总线短路 | 短路通信线 | 总线故障测试 |
| 错误帧注入 | 发送错误帧 | 错误处理测试 |
| 延迟注入 | 延迟信号传输 | 时序测试 |

### 3.2 软件故障注入

#### 3.2.1 数据注入

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 输入数据修改 | 修改输入参数值 | 数据有效性测试 |
| 边界值注入 | 注入边界值数据 | 边界处理测试 |
| 无效数据注入 | 注入无效/错误数据 | 错误处理测试 |
| 数据损坏注入 | 注入损坏的数据 | 数据完整性测试 |

#### 3.2.2 状态注入

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 非法状态注入 | 强制进入非法状态 | 状态机测试 |
| 状态跳转注入 | 跳过正常状态转换 | 状态转换测试 |
| 状态冲突注入 | 同时进入多个状态 | 状态冲突测试 |

#### 3.2.3 时序注入

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 延迟注入 | 延迟函数调用 | 时序测试 |
| 超时注入 | 强制触发超时 | 超时处理测试 |
| 时钟异常注入 | 修改系统时钟 | 时间相关测试 |

### 3.3 仿真故障注入

#### 3.3.1 HIL故障注入

| 方法 | 说明 | 优势 |
|------|------|------|
| 信号仿真故障 | 在HIL仿真中注入信号故障 | 可重复、安全 |
| 模型故障注入 | 在仿真模型中注入故障 | 可控性强 |
| 环境故障仿真 | 仿真环境条件故障 | 可模拟极端条件 |

#### 3.3.2 SIL故障注入

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 软件接口注入 | 在软件接口处注入故障 | 软件测试阶段 |
| 变量修改 | 直接修改内部变量 | 单元/集成测试 |
| 函数替换 | 替换函数实现 | 特定功能测试 |

## 四、故障注入测试设计

### 4.1 故障注入测试用例结构

```
故障注入测试用例:

用例编号: FI-[系统]-[故障类型]-[序号]
用例名称: [故障描述]

前置条件:
  - 系统状态: [正常工作状态]
  - 工况条件: [特定工况]

故障注入:
  - 故障类型: [硬件/软件/通信/环境]
  - 故障描述: [具体故障内容]
  - 注入方式: [注入方法]
  - 注入时机: [何时注入]
  - 持续时间: [故障持续时间]

预期响应:
  - 检测时间: ≤XXms
  - 故障码: [DTC码]
  - 系统行为: [降级/禁用/安全状态]
  - 用户提示: [仪表指示/警告]

后置验证:
  - 系统状态验证
  - 功能恢复验证
  - 故障码清除验证

测试环境:
  - 测试平台: [HIL/SIL/实车]
  - 测试工具: [工具名称]
```

### 4.2 故障注入测试设计步骤

1. **识别故障模式**
   - 分析系统架构
   - 识别所有可能的故障源
   - 参考FMEA/FMEDA结果

2. **确定故障注入点**
   - 确定注入位置
   - 确定注入方法
   - 评估注入可行性

3. **设计故障场景**
   - 确定前置条件
   - 确定注入时机
   - 确定故障持续时间

4. **定义预期响应**
   - 确定检测时间要求
   - 确定故障码
   - 确定系统行为
   - 确定用户提示

5. **设计验证方法**
   - 确定验证点
   - 确定验证标准
   - 确定数据采集方式

## 五、典型故障注入场景

### 5.1 传感器故障注入

#### 5.1.1 轮速传感器故障

```
用例编号: FI-ABS-SEN-001
用例名称: 左前轮速传感器信号丢失

前置条件:
  - 车速: 60 km/h
  - ABS系统正常工作
  - 路面: 干沥青

故障注入:
  - 故障类型: 硬件故障
  - 故障描述: 左前轮速传感器信号断开
  - 注入方式: 断开传感器信号线
  - 注入时机: 正常行驶中
  - 持续时间: 持续

预期响应:
  - 检测时间: ≤100ms
  - 故障码: DTC C0035
  - 系统行为: ABS功能降级，EBD保持
  - 用户提示: ABS故障灯亮起

后置验证:
  - ABS功能禁用
  - EBD功能正常
  - 故障码可读取
  - 故障恢复后系统正常
```

## 六、故障注入测试覆盖率

### 6.1 故障覆盖率要求

| ASIL等级 | 单点故障覆盖 | 潜伏故障覆盖 |
|----------|--------------|--------------|
| ASIL A | ≥60% | ≥60% |
| ASIL B | ≥90% | ≥60% |
| ASIL C | ≥97% | ≥90% |
| ASIL D | ≥99% | ≥97% |

### 6.2 故障注入覆盖率计算

```
故障覆盖率 = 已测试故障数 / 总故障数 × 100%

总故障数 = 硬件故障数 + 软件故障数 + 通信故障数 + 环境故障数
```

### 6.3 覆盖率验证清单

| 检查项 | 要求 |
|--------|------|
| 所有传感器故障是否覆盖 | 根据ASIL等级 |
| 所有执行器故障是否覆盖 | 根据ASIL等级 |
| 所有通信故障是否覆盖 | 根据ASIL等级 |
| 电源故障是否覆盖 | 要求 |
| 所有安全状态是否验证 | 要求 |
| 故障恢复是否验证 | 要求 |

### 7.3 工具选择考虑因素

1. **注入能力**：是否支持所需故障类型
2. **精度要求**：时间精度、数值精度
3. **可重复性**：故障注入的可重复性
4. **安全性**：是否可能损坏被测系统
5. **成本考虑**：工具成本、开发成本
