# ASIL等级测试要求

## 一、ASIL等级概述

ASIL（Automotive Safety Integrity Level，汽车安全完整性等级）是ISO 26262标准中定义的安全等级，分为四个等级：ASIL A、ASIL B、ASIL C、ASIL D。

| ASIL等级 | 安全完整性要求 | 典型应用 |
|----------|----------------|----------|
| ASIL A | 最低 | 非关键功能 |
| ASIL B | 低 | 一般安全功能 |
| ASIL C | 中 | 重要安全功能 |
| ASIL D | 高 | 关键安全功能 |

---

## 二、各ASIL等级测试要求

### 2.1 ASIL A 测试要求

**基本要求：**
- 功能正确性测试
- 基础故障检测测试

**测试方法：**
- 等价类划分
- 边界值分析
- 功能测试

**覆盖要求：**
- 分支覆盖（Branch Coverage）

**测试重点：**
1. 功能是否按规格说明正确执行
2. 输入参数边界值处理
3. 基本的错误处理能力

**测试用例示例：**
```
需求：灯光控制功能（ASIL A）

测试内容：
1. 正常功能测试
   - 开灯命令 → 灯光开启
   - 关灯命令 → 灯光关闭

2. 边界值测试
   - 最小亮度设置
   - 最大亮度设置

3. 错误处理测试
   - 无效命令处理
   - 通信超时处理
```

---

### 2.2 ASIL B 测试要求

**基本要求：**
- ASIL A所有要求
- 故障检测能力测试
- 故障响应时间测试

**测试方法：**
- 等价类划分
- 边界值分析
- 决策表测试
- 故障注入测试

**覆盖要求：**
- 分支覆盖（Branch Coverage）
- 故障注入覆盖

**测试重点：**
1. ASIL A所有测试内容
2. 故障检测能力验证
3. 故障响应时间验证
4. 故障码记录能力

**测试用例示例：**
```
需求：胎压监测功能（ASIL B）

测试内容：
1. 功能正确性测试
   - 正常胎压显示
   - 胎压异常报警

2. 故障检测能力测试
   - 传感器信号丢失检测
   - 传感器信号异常检测
   - 检测时间 ≤ 500ms

3. 故障响应测试
   - 故障后报警提示
   - 故障码正确记录
   - 故障恢复后正常工作

4. 故障注入测试
   - 模拟传感器断开
   - 模拟信号漂移
   - 模拟通信中断
```

---

### 2.3 ASIL C 测试要求

**基本要求：**
- ASIL B所有要求
- 冗余机制测试
- 安全状态验证

**测试方法：**
- 完整黑盒测试方法
- 故障注入测试
- 状态转换测试

**覆盖要求：**
- MC/DC覆盖（Modified Condition/Decision Coverage）
- 完整故障注入覆盖

**测试重点：**
1. ASIL B所有测试内容
2. 冗余机制验证
3. 安全状态进入验证
4. 安全状态维持验证
5. 故障恢复验证

**测试用例示例：**
```
需求：电子驻车制动EPB（ASIL C）

测试内容：
1. 功能正确性测试
   - 正常驻车制动施加
   - 正常驻车制动释放
   - 紧急制动功能

2. 故障检测能力测试
   - 电机故障检测
   - 位置传感器故障检测
   - 控制器故障检测
   - 检测时间 ≤ 100ms

3. 冗余机制测试
   - 主通道故障 → 备用通道工作
   - 双通道切换测试
   - 冗余数据比对测试

4. 安全状态测试
   - 故障后进入安全状态
   - 安全状态：驻车制动保持
   - 安全状态维持能力
   - 故障恢复后正常工作

5. 故障注入测试
   - 单点故障注入
   - 潜伏故障注入
   - 多点故障注入
```

---

### 2.4 ASIL D 测试要求

**基本要求：**
- ASIL C所有要求
- 完整安全机制测试
- 系统级安全验证

**测试方法：**
- 完整黑盒测试方法
- 完整故障注入测试
- 形式化验证（推荐）

**覆盖要求：**
- MC/DC覆盖
- 完整故障注入覆盖
- 100%安全需求覆盖

**测试重点：**
1. ASIL C所有测试内容
2. 完整安全机制验证
3. 系统级安全验证
4. 极端条件安全验证
5. 安全生命周期验证

**测试用例示例：**
```
需求：电子稳定控制ESC（ASIL D）

测试内容：
1. 功能正确性测试
   - 正常行驶稳定控制
   - 紧急情况稳定控制
   - 多种路面条件测试

2. 故障检测能力测试
   - 所有传感器故障检测
   - 所有执行器故障检测
   - 控制器故障检测
   - 检测时间 ≤ 50ms

3. 冗余机制测试
   - 传感器冗余验证
   - 处理器冗余验证
   - 通信冗余验证
   - 电源冗余验证

4. 安全状态测试
   - 故障后进入安全状态
   - 安全状态：ESC功能降级/禁用
   - 基础制动功能保持
   - 安全状态维持验证

5. 完整故障注入测试
   - 单点故障注入
   - 多点故障注入
   - 潜伏故障注入
   - 共因故障注入

6. 系统级安全验证
   - 整车安全验证
   - 极端工况安全验证
   - 环境极限安全验证
```

## 三、故障注入测试要求

### 3.1 故障类型与ASIL等级

| 故障类型 | ASIL A | ASIL B | ASIL C | ASIL D |
|----------|--------|--------|--------|--------|
| 单点故障 | 可选 | 要求 | 要求 | 要求 |
| 潜伏故障 | 可选 | 可选 | 要求 | 要求 |
| 多点故障 | 可选 | 可选 | 可选 | 要求 |
| 共因故障 | 可选 | 可选 | 可选 | 要求 |

### 3.2 故障检测时间要求

| ASIL等级 | 故障检测时间要求 |
|----------|------------------|
| ASIL A | 无明确要求 |
| ASIL B | ≤ 500ms |
| ASIL C | ≤ 100ms |
| ASIL D | ≤ 50ms |

### 3.3 故障响应要求

| ASIL等级 | 故障响应要求 |
|----------|--------------|
| ASIL A | 记录故障，功能可能降级 |
| ASIL B | 记录故障，功能降级或禁用 |
| ASIL C | 记录故障，进入安全状态 |
| ASIL D | 记录故障，立即进入安全状态 |

## 四、测试覆盖要求

### 4.1 代码覆盖要求

| ASIL等级 | 语句覆盖 | 分支覆盖 | MC/DC覆盖 |
|----------|----------|----------|-----------|
| ASIL A | 要求 | 推荐 | 可选 |
| ASIL B | 要求 | 要求 | 推荐 |
| ASIL C | 要求 | 要求 | 要求 |
| ASIL D | 要求 | 要求 | 要求 |

### 4.2 需求覆盖要求

| ASIL等级 | 需求覆盖要求 |
|----------|--------------|
| ASIL A | 所有功能需求覆盖 |
| ASIL B | 所有功能需求 + 安全需求覆盖 |
| ASIL C | 所有需求100%覆盖 |
| ASIL D | 所有需求100%覆盖 + 安全需求专项覆盖 |

### 4.3 接口覆盖要求

| ASIL等级 | 接口覆盖要求 |
|----------|--------------|
| ASIL A | 主要接口覆盖 |
| ASIL B | 所有外部接口覆盖 |
| ASIL C | 所有接口覆盖 + 异常测试 |
| ASIL D | 所有接口覆盖 + 完整异常测试 |
