# 测试设计技术详细说明

## 一、等价类划分法

### 1.1 基本概念

等价类划分法是一种典型的黑盒测试方法，其核心思想是：将输入域划分为若干个等价类，每个等价类中的测试数据对于揭露程序错误是等效的，因此只需从每个等价类中选取一个代表值进行测试。

### 1.2 等价类类型

**有效等价类：**
- 对于程序规格说明而言，是合理的、有意义的输入数据集合
- 用于验证程序是否实现了规格说明中规定的功能和性能

**无效等价类：**
- 对于程序规格说明而言，是不合理的、无意义的输入数据集合
- 用于验证程序对无效输入的处理能力

### 1.3 划分原则

1. **区间划分**：如果输入条件规定了取值范围，则可以确定一个有效等价类和两个无效等价类
2. **枚举划分**：如果输入条件规定了枚举值，则每个枚举值是一个有效等价类，所有非枚举值构成一个无效等价类
3. **布尔划分**：如果输入条件是布尔值，则true和false各是一个有效等价类
4. **条件划分**：如果输入条件规定了必须满足的条件，则可以确定一个有效等价类和一个无效等价类

### 1.4 测试用例设计步骤

1. 确定等价类表，列出所有划分出的等价类
2. 为每个等价类规定一个唯一编号
3. 设计一个新的测试用例，使其尽可能多地覆盖尚未被覆盖的有效等价类
4. 重复步骤3，直到所有有效等价类都被覆盖
5. 设计新的测试用例，使其仅覆盖一个尚未被覆盖的无效等价类
6. 重复步骤5，直到所有无效等价类都被覆盖

### 1.5 车载应用示例

**示例：车速参数测试**

```
需求描述：系统接收车速信号，范围0-200 km/h

等价类划分：

|------|------|------|--------|
| EC1 | 有效 | 静止状态 | 0 |
| EC2 | 有效 | 低速行驶 | 15 |
```

## 二、边界值分析法

### 2.1 基本概念

边界值分析法是对等价类划分法的补充，其核心思想是：错误往往发生在输入范围的边界上，因此应该在边界附近选取测试数据。

### 2.2 边界值类型

**标准边界值：**
- 最小值
- 最小值+1（或略大于最小值）
- 典型值
- 最大值-1（或略小于最大值）
- 最大值

**健壮边界值：**
- 最小值-1（或略小于最小值）
- 最小值
- 最小值+1
- 典型值
- 最大值-1
- 最大值
- 最大值+1（或略大于最大值）

### 2.3 边界值选取原则

1. 如果输入条件规定了取值范围，则应选取边界值及边界附近的值
2. 如果输入条件规定了值的个数，则应选取最小个数、最大个数、比最小个数少1、比最大个数多1的值
3. 对于输出条件，同样应用边界值分析
4. 如果程序规格说明中提到的边界值，一定要作为测试数据

## 三、决策表测试法

### 3.1 基本概念

决策表测试法是一种针对多条件组合逻辑的测试方法，其核心思想是：系统地列出所有可能的条件组合及其对应的动作，确保每种组合都被测试到。

### 3.2 决策表结构

决策表由四个部分组成：

1. **条件桩**：列出所有条件
2. **动作桩**：列出所有可能的动作
3. **条件项**：列出每个条件的取值
4. **动作项**：列出每种条件组合对应的动作

### 3.3 决策表构建步骤

1. 列出所有条件
2. 列出所有可能的动作
3. 确定每种条件组合对应的动作
4. 简化决策表（合并等价规则）
5. 为每条规则设计测试用例

### 3.4 决策表简化规则

1. 如果某条件对动作没有影响，可以用"-"表示不关心
2. 如果多条规则的动作相同，且某些条件取值相同，可以合并
3. 如果某条件对所有动作都没有影响，可以从表中删除

## 四、状态转换测试法

### 4.1 基本概念

状态转换测试法是一种针对状态机系统的测试方法，其核心思想是：基于系统的状态转换模型，测试状态转换的正确性。

### 4.2 状态转换模型

状态转换模型由以下元素组成：

1. **状态**：系统在某一时刻所处的模式或条件
2. **事件**：触发状态转换的条件
3. **转换**：从一个状态到另一个状态的变化
4. **动作**：状态转换时执行的操作

### 4.3 覆盖准则

| 准则 | 定义 | 强度 | 最小用例数 |
|------|------|------|------------|
| 状态覆盖 | 每个状态至少访问一次 | ★★☆☆☆ | ≥ 状态数 |
| 转换覆盖 | 每个转换至少执行一次 | ★★★☆☆ | ≥ 转换数 |
| 路径覆盖 | 所有可能路径都执行 | ★★★★★ | 所有可能路径 |

### 4.4 状态转换测试步骤

1. 识别所有状态
2. 识别所有转换事件
3. 绘制状态转换图
4. 生成状态转换表
5. 选择覆盖准则
6. 设计测试用例

### 4.5 车载应用示例

**示例：TCS系统状态转换测试**

```
需求描述：
- 车辆启动后，TCS进入监控状态
- 检测到打滑时，TCS激活
- 打滑消除后，TCS退出激活
- 检测到故障时，进入故障状态

状态定义：
  S1: 待机状态（系统未启动）

状态转换表：

| TC1 | S1→E1→S2→E6→S1 | S1, S2 |
```

## 五、场景测试法

### 5.1 基本概念

场景测试法是一种基于用户实际使用场景的测试方法，其核心思想是：模拟用户的实际操作流程，测试系统在真实场景下的行为。

### 5.2 场景类型

1. **基本场景**：正常用户操作流程
2. **替代场景**：替代操作路径
3. **异常场景**：错误处理流程
4. **边界场景**：极限条件场景

### 5.3 场景描述格式

```
场景名称: [场景描述]

前置条件:
  - 条件1
  - 条件2

测试步骤:
  步骤1: [操作] → [预期结果]
  步骤2: [操作] → [预期结果]
  ...

后置条件:
  - 结果状态1
  - 结果状态2
```

### 5.4 车载应用示例

**示例：ABS紧急制动场景测试**

```
场景1: 正常紧急制动（基本场景）

前置条件:
  - 车辆处于正常运行状态
  - ABS系统无故障
  - 路面为干沥青

测试步骤:
  步骤1: 驾驶员启动车辆 → 车辆正常启动
  步骤2: 车辆加速至60km/h → 车速稳定
  步骤3: 驾驶员快速踩下制动踏板 → ABS激活
  步骤4: 保持制动直至停止 → 车辆平稳停止

预期结果:
  - ABS指示灯短暂亮起后熄灭
  - 踏板有回弹感
  - 车辆保持直线
  - 制动距离≤40m

---

场景2: 低速制动不激活ABS（替代场景）

前置条件:
  - 车辆处于正常运行状态
  - ABS系统无故障

测试步骤:
  步骤1: 驾驶员启动车辆 → 车辆正常启动
  步骤2: 车辆加速至20km/h → 车速稳定
  步骤3: 驾驶员快速踩下制动踏板 → ABS不激活
  步骤4: 车辆正常制动停止 → 车辆停止

  - ABS指示灯不亮
  - 正常制动感觉
  - 车辆保持直线

---

场景3: ABS故障时制动（异常场景）

前置条件:
  - ABS系统存在故障
  - ABS故障指示灯亮起

测试步骤:
  步骤1: 驾驶员启动车辆 → ABS故障灯常亮
  步骤2: 车辆加速至60km/h → 车速稳定
  步骤3: 驾驶员快速踩下制动踏板 → 基础制动工作
  步骤4: 车辆停止 → 车辆停止

  - ABS功能禁用
  - 基础制动正常工作

---

场景4: 冰雪路面紧急制动（边界场景）

前置条件:
  - 车辆在冰雪路面行驶
  - ABS系统无故障
  - 路面附着系数约0.2

测试步骤:
  步骤1: 车辆加速至40km/h → 车速稳定
  步骤2: 驾驶员紧急制动 → ABS激活
  步骤3: ABS持续调节 → 车轮不抱死
  步骤4: 车辆停止 → 车辆停止

预期结果:
  - ABS持续工作
  - 车轮不抱死
  - 车辆方向可控
  - 制动距离延长但稳定
```

## 六、测试方法选择指南

### 6.1 根据需求类型选择

| 需求类型 | 首选方法 | 补充方法 | 说明 |
|----------|----------|----------|------|
| 功能需求 | 等价类划分 + 边界值分析 | 决策表、状态转换 | 覆盖功能点、边界条件 |
| 性能需求 | 边界值分析 | 负载测试、压力测试 | 关注性能边界 |
| 接口需求 | 等价类划分 | 接口一致性测试 | 覆盖接口参数 |
| 安全需求 | 故障注入测试 | 状态转换、场景测试 | 覆盖安全场景 |
| 状态机需求 | 状态转换测试 | 场景测试 | 覆盖状态转换 |

### 6.2 根据ASIL等级选择

| ASIL等级 | 推荐方法组合 | 覆盖要求 |
|----------|--------------|----------|
| ASIL_A | 等价类 + 边界值 | 分支覆盖 |
| ASIL_B | 等价类 + 边界值 + 决策表 + 故障注入 | 分支覆盖 |
| ASIL_C | 完整黑盒 + 故障注入 | MC/DC覆盖 |
| ASIL_D | 完整黑盒 + 完整故障注入 | MC/DC覆盖 |

### 6.3 根据测试阶段选择

| 测试阶段 | 推荐方法 | 说明 |
|----------|----------|------|
| 单元测试 | 边界值、等价类 | 关注函数/模块级别 |
| 集成测试 | 接口测试、场景测试 | 关注模块间交互 |
| 系统测试 | 完整黑盒方法 | 关注系统功能 |
| 验收测试 | 场景测试 | 关注用户场景 |
